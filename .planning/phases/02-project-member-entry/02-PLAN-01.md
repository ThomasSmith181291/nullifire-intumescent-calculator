    5. **app/services/calc_service.py** — Pure calculation functions:
       - `calculate_member_quantities(dft_mm, heated_perimeter_mm, length_m, quantity, solid_factor, density)` -> dict with surface_area_m2, volume_litres, weight_kg.
       - Formula: surface_area_m2 = (heated_perimeter_mm / 1000) * length_m * quantity. volume_litres = surface_area_m2 * (dft_mm / 1000) / solid_factor * 1000. weight_kg = volume_litres * density.
       - If dft_mm is None or 0, return all zeros.
       - Round surface_area_m2 to 3 decimal places, volume_litres to 2, weight_kg to 2.

    6. **app/services/project_service.py** — Project and member CRUD:
       - `create_project(name, product_id, fire_rating_id, failure_temp_id, client='', reference='', origin_id=1)`: creates .nfc SQLite file in PROJECT_DB_DIR named `{uuid}.nfc`, inserts project row, returns project dict.
       - `get_project(project_id)`: reads project row + all members, returns dict with project fields and `members` list.
       - `update_project(project_id, **kwargs)`: updates project fields, returns updated project dict.
       - `list_projects()`: scans PROJECT_DB_DIR for .nfc files, reads project name from each, returns list of {id, name, client, updated_at}.
       - `add_member(project_id, section_id, hp_profile_name, quantity=1, length_m=0, zone='', level='', fire_rating_id=None, failure_temp_id=None, product_id=None)`: 
         (a) Get section info via section_service.get_section(section_id) for section_name and steel_type.
         (b) Resolve effective product/rating/temp (member override or project default).
         (c) Call dft_service.lookup_dft() to get hp_over_a, heated_perimeter, dft_mm, final_dft_mm, status.
         (d) Get product via product_service.get_product() for solid_factor and density.
         (e) Call calc_service.calculate_member_quantities() for surface_area, litres, weight.
         (f) INSERT member row with all computed fields. Return member dict.
       - `update_member(project_id, member_id, **kwargs)`: updates specified fields, re-runs DFT+calc if any of section_id, hp_profile_name, fire_rating_id, failure_temp_id, product_id, quantity, or length_m changed. Returns updated member dict.
       - `delete_member(project_id, member_id)`: DELETE single member.
       - `delete_members_batch(project_id, member_ids)`: DELETE WHERE id IN (...) for batch delete (per ENTRY-10).
       - `_recalculate_member(project_id, member_row)`: internal helper that runs DFT lookup + calc for a member, returns computed fields dict.

       ```
       POST   /api/projects                          — Create project
         Body: {name, product_id, fire_rating_id, failure_temp_id, client?, reference?, origin_id?}
         Returns: 201 + project dict

       GET    /api/projects                           — List all projects
         Returns: [{id, name, client, updated_at}, ...]

       GET    /api/projects/<project_id>              — Get project with members
         Returns: {id, name, ..., members: [{id, section_name, dft_mm, ...}, ...]}

       PUT    /api/projects/<project_id>              — Update project settings
         Body: {name?, product_id?, fire_rating_id?, failure_temp_id?, origin_id?, client?, reference?}
         Returns: updated project dict
         NOTE: If product_id, fire_rating_id, or failure_temp_id changed, recalculate ALL members
               that use the project default (where their override is NULL). Return project with updated members.

       DELETE /api/projects/<project_id>              — Delete project (removes .nfc file)
         Returns: 204

       POST   /api/projects/<project_id>/members      — Add member
         Body: {section_id, hp_profile_name, quantity?, length_m?, zone?, level?,
                fire_rating_id?, failure_temp_id?, product_id?}
         Returns: 201 + member dict with all computed fields

       PUT    /api/projects/<project_id>/members/<member_id>  — Update member
         Body: any member fields to update
         Returns: updated member dict (recalculated if needed)

       DELETE /api/projects/<project_id>/members/<member_id>  — Delete single member
         Returns: 204

       POST   /api/projects/<project_id>/members/batch-delete — Batch delete
         Body: {ids: [member_id, ...]}
         Returns: 204
       ```
