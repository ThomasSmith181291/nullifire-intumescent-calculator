**static/js/api.js** — API client module (ES6 export):
- `api.createProject(data)` — POST /api/projects
- `api.getProject(id)` — GET /api/projects/{id}
- `api.updateProject(id, data)` — PUT /api/projects/{id}
- `api.listProjects()` — GET /api/projects
- `api.deleteProject(id)` — DELETE /api/projects/{id}
- `api.addMember(projectId, data)` — POST /api/projects/{id}/members
- `api.updateMember(projectId, memberId, data)` — PUT /api/projects/{id}/members/{mid}
- `api.deleteMember(projectId, memberId)` — DELETE /api/projects/{id}/members/{mid}
- `api.batchDeleteMembers(projectId, ids)` — POST /api/projects/{id}/members/batch-delete
- `api.searchSections(q, origin)` — GET /api/sections/search?q=...
- `api.getSectionProfiles(sectionId, productId)` — GET /api/sections/{id}/profiles
- `api.getSectionFactor(sectionId, profile)` — GET /api/sections/{id}/factor
- `api.getProducts()` — GET /api/products
- `api.getProductFireRatings(productId)` — GET /api/products/{id}/fire-ratings
- `api.getProductFailureTemps(productId, fireRatingId)` — GET /api/products/{id}/failure-temps
- `api.getOrigins()` — GET /api/origins
- `api.lookupDft(data)` — POST /api/dft/lookup
- All functions return fetch promises. Base URL configurable.

**static/js/sidebar.js** — Sidebar project settings module:
- `initSidebar()` — populates product, fire rating, temp, origin dropdowns from API
- `loadProject(projectId)` — fills sidebar form from project data
- `onProjectSettingsChange(callback)` — registers change handler
- New project button: opens create dialog (name, client, ref), calls api.createProject
- Load project: shows list from api.listProjects
- Product dropdown change: cascades to update fire ratings, then failure temps
- All changes auto-save via api.updateProject
