<objective>
Implement the section typeahead cell editor and member add/edit/delete flows in AG Grid, with keyboard-driven entry and smart defaults.

Purpose: This is where the "fast" in "fast member entry" happens. Users type a section name, get instant filtered results, select one, and the exposure auto-defaults. Tab through remaining fields, Enter to add.
Output: Users can rapidly add steel members entirely by keyboard with typeahead search completing in under 50ms.
</objective>

**static/js/section-editor.js** — Custom AG Grid cell editor for section typeahead:
- Implements AG Grid ICellEditorComp interface (init, getGui, getValue, destroy, afterGuiAttached)
- Creates a container div with an input field and a dropdown results list
- On input change (debounced 100ms): call api.searchSections(value, currentOrigin)
- Display results as clickable list items showing: serial_size | steel_type_abbrev | weight kg/m
- Arrow keys navigate results, Enter selects, Escape cancels
- On select: store the full section object, return section_id as the cell value
- Input must handle partial matches: typing "254" shows all 254x... sections
- Must work within AG Grid's cell editing lifecycle (startEditing, stopEditing)

**static/js/grid-manager.js** — Grid management module:
- `initGrid(gridDiv, projectId)` — creates AG Grid with full column config
- Column definitions with cell editors:
  - Section: custom SectionEditor, valueFormatter shows serial_size
  - Type: read-only (auto-set when section selected)
  - Exposure: agSelectCellEditor with values from section profiles
  - Fire Rating: agSelectCellEditor with values from product fire ratings
  - Temp: agSelectCellEditor with values from product failure temps
  - Qty: agNumberCellEditor, default 1
  - Length: agNumberCellEditor, default 0
  - Zone: agTextCellEditor
  - Level: agTextCellEditor
  - Computed columns (Hp/A, DFT, Area, Litres, Status): not editable
- `addEmptyRow()` — adds a pinned bottom row for new entry
- `onCellValueChanged(event)` — handler for all cell edits:
  - If section changed: fetch profiles, auto-set default exposure (ENTRY-03), clear computed fields
  - If exposure changed: clear DFT-related computed fields
  - If any editable field changed: call api.updateMember with changed fields, update row with response
- `deleteSelectedRows()` — get selected rows, call api.batchDeleteMembers
- Tab flow configuration: tabToNextCell callback that skips computed columns (ENTRY-07)
- Keyboard handler: Enter on last editable column of pinned row = save member and add new row

**static/js/app.js** — Update to wire grid-manager with sidebar:
- On project load: call gridManager.initGrid with project members
- On sidebar settings change: refresh grid data (recalculated members from API)
- Add "Delete Selected" button/keyboard shortcut
- Wire addEmptyRow to a "+" button or keyboard shortcut
