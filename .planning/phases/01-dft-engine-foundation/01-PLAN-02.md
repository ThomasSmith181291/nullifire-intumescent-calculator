**app/services/section_service.py:**

All functions take plain Python args, return dicts/lists. No Flask request objects.

`search_sections(query: str, origin_id: int = None, limit: int = 50) -> list[dict]`:
- Query `steel_sections` JOIN `steel_types` JOIN `origins`
- WHERE `serial_size LIKE '%{query}%'` (parameterized!)
- If origin_id provided, filter by `origin_id = ?`
- ORDER BY `steel_type_id, group_sort, weight DESC`
- LIMIT to `limit` (default 50, max 100)
- Return list of dicts: `{id, serial_size, depth, width, web_thickness, flange_thickness, weight, area, steel_type_id, steel_type_name, steel_type_abbrev, origin_id}`

`get_section(section_id: int) -> dict | None`:
- Get single section by ID with steel type info
- Return dict or None if not found

`get_section_profiles(section_id: int, product_id: int = None) -> list[dict]`:
- First get the section's `steel_type_id`
- Query `hp_profiles` WHERE `steel_type_id = section.steel_type_id`
- Filter OUT `board_only = 1` (intumescent products don't use board profiles)
- If `product_id` provided, filter to profiles that exist in `catalogue` WHERE `band = product.catalogue_band`
- For each profile, check if `section_factors` has a non-null row for `(section_id, hp_profiles.hp_over_a_band)` -- only return profiles with valid section factor data
- Return list of dicts: `{name, description, abbreviation, faces, hp_over_a_band, is_composite}`

`get_section_factor(section_id: int, hp_profile_name: str) -> dict | None`:
- Look up `hp_profiles` to get `hp_over_a_band` for the profile
- Query `section_factors` WHERE `steel_section_id = ? AND hp_over_a_band = ?`
- Return `{hp_over_a, heated_perimeter, hp_over_a_band}` or None

Register routes on `api_bp` (import from `app.api`).

`GET /api/sections/search?q=<query>&origin=<origin_id>&limit=<limit>`:

`GET /api/sections/<int:section_id>`:

`GET /api/sections/<int:section_id>/profiles?product_id=<product_id>`:

`GET /api/sections/<int:section_id>/factor?profile=<hp_profile_name>`:
- Call `section_service.get_section_factor()`
- Return JSON dict with `{hp_over_a, heated_perimeter}`. 400 if profile missing, 404 if no factor found.

**app/services/product_service.py:**

`get_products() -> list[dict]`:
- Query `products` WHERE `is_discontinued = 0` (active products only)
- Return list of dicts: `{id, name, density, solid_factor, offset_dft, catalogue_band, container_1_litres, container_2_litres, container_3_litres, container_4_litres, container_1_kg, container_2_kg, container_3_kg, container_4_kg, is_solvent_based, description}`
- ORDER BY `name`
- The 5 expected products: SC601 (278), SC801-120 (283), SC802 (266), SC804 (324), SC901 (284)

`get_product(product_id: int) -> dict | None`:
- Single product by ID with all fields above

`get_product_fire_ratings(product_id: int) -> list[dict]`:
- Query `fire_ratings` filtered to ratings that have DFT data for this product:
  `SELECT DISTINCT fr.* FROM fire_ratings fr JOIN dft_data d ON d.fire_rating_id = fr.id WHERE d.product_id = ?`
- Return list of dicts: `{id, description, abbrev, short_abbrev}`
- ORDER BY `id` (ascending = shortest to longest fire period)

`get_product_failure_temps(product_id: int, fire_rating_id: int) -> list[dict]`:
- Query `failure_temps` filtered to temps that have DFT data for this product AND fire rating:
  `SELECT DISTINCT ft.* FROM failure_temps ft JOIN dft_data d ON d.failure_temp_id = ft.id WHERE d.product_id = ? AND d.fire_rating_id = ?`
- Exclude meta-entries: filter out `master_ft_id` that don't map to actual temperatures (id=1 'BS 476' and id=12 'Multi' are meta -- only return entries where `is_country_specific = 1`)
- Return list of dicts: `{id, description, temp_id, master_ft_id}`
- ORDER BY `id`

`get_origins() -> list[dict]`:
- Query all origins
- Return `{id, code, description}`
