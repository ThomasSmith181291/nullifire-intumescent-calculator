**run.py:**
- Import `webview`, `threading`, `time` from stdlib, and `create_app` from app
- `start_flask(app, ready_event)`:
  - Call `ready_event.set()` BEFORE `app.run()` -- no, actually set it just before run starts. Use a small delay or Flask's `before_first_request` alternative. Simplest: set the event, then run.
  - Actually, the correct pattern per STACK.md: `ready_event.set()` then `app.run(host='127.0.0.1', port=5000, debug=False, use_reloader=False)`
  - Flask starts serving immediately after `app.run()` is called, but the event should be set just before so PyWebView doesn't race. Use this pattern:
    ```python
    def start_flask(app, ready_event):
        ready_event.set()
        app.run(host='127.0.0.1', port=5000, debug=False, use_reloader=False)
    ```
- `if __name__ == '__main__':` block:
  - `app = create_app()`
  - `ready_event = threading.Event()`
  - Start Flask in daemon thread: `threading.Thread(target=start_flask, args=(app, ready_event), daemon=True).start()`
  - `ready_event.wait(timeout=5)` -- wait for Flask to be ready
  - `webview.create_window('Nullifire Intumescent Calculator', 'http://127.0.0.1:5000', width=1400, height=900, min_size=(1024, 700))`
  - `webview.start()`
- Also support running Flask standalone for browser development:
  - Add a `--dev` flag check: `if '--dev' in sys.argv: app.run(host='127.0.0.1', port=5000, debug=True)`
  - This lets developers work in a regular browser without PyWebView
